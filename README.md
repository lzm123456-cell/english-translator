# 英文翻译助手 — 读英文文章，像读中文一样快 🇨🇳⚡

> **选中即看，零延迟。** 不用切换页面，不用复制粘贴，不用等 API 慢慢返回。

---

## 它解决的问题

你读英文文章的时候是不是这样？

- 看到一个不错的英文长文 → 打开
- 读两段遇到生词 → 复制 → 打开翻译网站 → 粘贴 → 看翻译 → 切回来
- 又遇到一个 → 再复制 → 再粘贴 → 再切回来
- **然后你就不想读了**

一些翻译插件又慢又卡，因为每次选中都要调一次 API，要等 3-5 秒才能看到结果。

**这个插件不一样：它只翻一次，后面全部零延迟。**

---

## 怎么用

**一步设置，简单粗暴：**

1. 打开一篇英文文章
2. 点右上角插件图标
3. 输入你的 DeepSeek API Key
4. 点「翻译本文」

**然后你就可以：**

| 动作 | 效果 |
|------|------|
| 选中一段文字 | 自动显示译文 |
| 按 **J** 键 | 切换显示/隐藏译文 |
| 按 **ESC** | 关闭译文 |
| 向下滚动 | 新段落自动补翻，无需等待 |

**全程不需要离开文章页面。**

---

## 跟其他翻译插件有什么区别

| | 传统翻译插件 | 英文翻译助手 |
|---|---|---|
| 每次选中 | 调一次 API（等 3-5 秒） | **缓存命中，零延迟** |
| 整篇阅读 | 翻一段等一段 | 一次翻完，后面秒出 |
| API 费用 | 翻 50 段 = 50 次调用 | 翻 50 段 = **1 次调用** |
| 阅读体验 | 频繁被打断 | 沉浸式不间断 |

**关键差异：** 它用的是一种 "预翻译" 架构——打开文章时一次把全文翻译好，你选中什么就从缓存里取，永远不用等。

## 安装方式

```bash
# 1. 下载本项目
git clone https://github.com/lzm123456-cell/english-translator.git

# 2. 打开 Chrome 扩展管理
chrome://extensions/

# 3. 打开右上角「开发者模式」

# 4. 点击「加载已解压的扩展程序」
#    选择 extension/ 目录

# 5. 打开任意英文文章，点插件图标输入 API Key，开始用
```

## 技术架构（如果你是开发者）

```
打开文章 → 一次 API 调用预翻译全文 → 构建原文↔译文映射表
  ↓
选中段落按 J → 缓存模糊匹配 → 零延迟显示
  ↓
新段落滚动到 → 后台静默补翻，无需手动操作
```

- Chrome Extension Manifest V3
- DeepSeek API（v4 模型，性价比极高）
- 纯前端，无后端，无数据离开你的电脑
- 开源，MIT 协议，你可以改
