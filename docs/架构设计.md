# 架构设计文档

## 整体架构

```
┌─────────────────────────────────────────────────────┐
│                   浏览器插件                          │
│  ┌──────────┐    ┌──────────────┐  ┌─────────────┐ │
│  │ content.js│◄──►│ background.js│◄─►│ popup.html  │ │
│  │ (页面内)  │    │ (后台/API)   │  │ (设置弹窗)  │ │
│  └─────┬────┘    └──────┬───────┘  └─────────────┘ │
│        │                │                            │
│  ┌─────▼────┐    ┌──────▼───────┐                    │
│  │ 选中+J键 │    │ DeepSeek API │                    │
│  │ 悬浮框   │    │ + 缓存管理   │                    │
│  └──────────┘    └──────────────┘                    │
└─────────────────────────────────────────────────────┘
```

## 核心模块

### 1. content.js — 页面交互
- 监听鼠标选中事件
- 监听 J 键按下（仅在选中文本时触发）
- 在选中位置旁创建/关闭悬浮框
- 从缓存查询译文，匹配最近段落

### 2. background.js — 后台逻辑
- 接收 content.js 的翻译请求
- 调用 DeepSeek API 分批翻译
- 管理翻译缓存（chrome.storage.local）
- 增量翻译（只翻未缓存段落）

### 3. popup.html/js — 用户配置
- API Key 设置
- 触发预翻译按钮
- 查看缓存状态
- 术语表管理

## 数据流

### 预翻译流程
1. content.js 提取页面可见文本段落
2. 发送段落列表到 background.js
3. background.js 检查缓存，过滤未翻译段落
4. 分批调用 DeepSeek API（每批 5-10 段）
5. 逐批写入 chrome.storage.local
6. 通知 content.js 进度

### 查译流程
1. 用户选中文本，按 J 键
2. content.js 获取选中文本
3. 在本地缓存中模糊匹配最近段落
4. 在选中位置显示悬浮框
5. 再按 J / 点击页面其它位置 → 关闭
